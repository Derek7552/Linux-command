# Linux特殊符号快速参考

## 📋 常用特殊符号总结

### 路径符号

| 符号 | 含义 | 示例 | 说明 |
|------|------|------|------|
| `~` | 用户主目录 | `cd ~` | 相当于 `/home/用户名` |
| `~user` | 指定用户主目录 | `cd ~root` | root用户的主目录 |
| `.` | 当前目录 | `./script.sh` | 当前目录下的script.sh |
| `..` | 上级目录 | `cd ..` | 切换到上级目录 |
| `-` | 上一个目录 | `cd -` | 切换到上一个工作目录 |
| `/` | 路径分隔符 | `/home/user` | 绝对路径 |

### 通配符

| 符号 | 含义 | 示例 | 匹配结果 |
|------|------|------|----------|
| `*` | 任意字符(0或多个) | `*.txt` | 所有.txt文件 |
| `?` | 单个字符 | `file?.txt` | file1.txt, file2.txt |
| `[abc]` | 字符集合 | `[abc].txt` | a.txt, b.txt, c.txt |
| `[0-9]` | 数字范围 | `file[0-9].txt` | file0.txt到file9.txt |
| `[!abc]` | 排除字符 | `[^abc]*` | 不以a,b,c开头的文件 |
| `{a,b,c}` | 花括号扩展 | `file{1,2,3}.txt` | file1.txt file2.txt file3.txt |

### 引用和转义

| 符号 | 含义 | 示例 | 输出结果 |
|------|------|------|----------|
| `"双引号"` | 允许变量扩展 | `"Hello $name"` | Hello John |
| `'单引号'` | 字面含义 | `'Hello $name'` | Hello $name |
| `\` | 转义字符 | `\$var` | 字面意义的$var |

## 🚀 常用组合模式

### 文件路径操作
```bash
~/Documents          # 主目录的Documents文件夹
./script.sh          # 当前目录的脚本
../data/             # 上级目录的data文件夹
../../config/        # 上上级目录的config文件夹
```

### 文件批量操作
```bash
*.log                # 所有日志文件
backup_*.tar.gz      # 所有备份压缩包
file[1-3].txt        # file1.txt, file2.txt, file3.txt
{jpg,png,gif}        # 三种图片格式
```

### 目录导航
```bash
cd ~                 # 回主目录
cd -                 # 回上个目录
cd ..                # 上一级
cd ../..             # 上上级
```

## 💡 使用技巧

### 1. 路径构造
- **绝对路径**：以 `/` 开头，如 `/home/user/file.txt`
- **相对路径**：不以 `/` 开头，如 `Documents/file.txt`

### 2. 通配符优先级
- `*` 最常用，匹配任意长度
- `?` 用于固定长度文件名
- `[]` 用于精确字符匹配
- `{}` 用于生成多个变体

### 3. 转义时机
- 在双引号内使用 `\` 转义 `$`, `` ` ``, `"`, `\`
- 在单引号内无需转义，保持字面含义
- 在命令行直接使用 `\` 转义shell特殊字符

### 4. 特殊情况
- `~+` 等同于 `pwd`（当前目录）
- `~-` 等同于 `cd -` 的目标目录（上一个目录）
- `/*` 匹配根目录下所有文件（需要小心使用）

## 🔧 实际应用示例

### 文件管理
```bash
# 备份所有配置文件
cp ~/.config/* ~/backup/config/

# 删除所有临时文件
rm /tmp/temp_*.tmp

# 移动图片文件
mv ~/Downloads/*.{jpg,png,gif} ~/Pictures/
```

### 系统操作
```bash
# 查看系统日志
tail -f /var/log/syslog

# 编辑用户配置文件
vim ~/.bashrc

# 运行上级目录的脚本
../scripts/deploy.sh
```

### 批量处理
```bash
# 重命名文件
for file in file?.txt; do mv "$file" "new_$file"; done

# 统计各类型文件数量
ls -1 *.{c,h,py,sh} | wc -l

# 创建多层目录结构
mkdir -p project/{src,docs/{api,user},tests}
```

## ⚠️ 注意事项

1. **通配符的潜在危险**：
   ```bash
   rm *         # 删除当前目录所有文件！
   rm -rf /*    # 删除整个系统！！！
   ```

2. **路径符号的上下文**：
   - `~` 只在交互式shell中有效
   - `.` 和 `..` 在所有路径上下文中有效
   - `-` 只在 `cd` 命令中有效

3. **引用选择**：
   - 需要变量扩展时用双引号
   - 需要字面字符串时用单引号
   - 处理包含空格的文件名时必须引用

记住：特殊符号是Linux命令行的灵魂，熟练掌握它们能让你的操作事半功倍！
