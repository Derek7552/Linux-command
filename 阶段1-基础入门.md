## 📋 学习内容

### 1.0 Linux特殊符号详解

在Linux命令行中，有一些特殊符号具有特定的含义，掌握这些符号的使用规律可以大大提高操作效率。

#### 路径相关符号

**`~` (波浪号) - 用户主目录**
- 含义：代表当前用户的主目录（通常是 `/home/用户名`）
- 使用规律：
  - `cd ~` 或 `cd`：切换到用户主目录
  - `~/Documents`：表示主目录下的Documents文件夹
  - `~user`：表示指定用户的主目录（如 `~root` 表示root用户的主目录）
- 示例：
  ```bash
  cd ~                    # 切换到自己的主目录
  ls ~/Downloads         # 列出主目录下的Downloads文件夹
  cp file.txt ~/backup/  # 复制文件到主目录的backup文件夹
  ```

**`.` (单点) - 当前目录**
- 含义：代表当前工作目录
- 使用规律：
  - `./program`：运行当前目录下的program文件
  - `ls .`：显示当前目录内容（相当于 `ls`）
  - `cp ./file.txt ./backup/`：从当前目录复制到当前目录的子目录
- 示例：
  ```bash
  ./script.sh           # 运行当前目录下的脚本
  ls -la .              # 显示当前目录的详细信息
  mv file.txt .         # 将file.txt移动到当前目录（通常无实际效果）
  ```

**`..` (双点) - 上级目录**
- 含义：代表当前目录的上级目录（父目录）
- 使用规律：
  - `cd ..`：切换到上级目录
  - `../file.txt`：上级目录中的file.txt文件
  - `../../`：上上级目录
  - `../../../`：上上上级目录
- 示例：
  ```bash
  cd ..                 # 返回上级目录
  cp ../data.txt .      # 从上级目录复制data.txt到当前目录
  ls ../../             # 显示上上级目录的内容
  ```

**`-` (连字符) - 上一个目录**
- 含义：在 `cd` 命令中代表上一个工作目录
- 使用规律：
  - `cd -`：快速切换回上一个工作目录
  - 这个符号只能在 `cd` 命令中使用
- 示例：
  ```bash
  pwd                   # /home/user/Documents
  cd /var/log          # 切换到/var/log
  cd -                 # 切换回/home/user/Documents
  ```

#### 路径分隔符

**`/` (斜杠) - 路径分隔符**
- 含义：Linux中的路径分隔符（不同于Windows的`\`）
- 使用规律：
  - 绝对路径：以 `/` 开头，表示从根目录开始的完整路径
  - 相对路径：不以 `/` 开头，表示相对于当前目录的路径
  - 连续的 `//` 等同于单个 `/`
- 示例：
  ```bash
  /home/user/file.txt    # 绝对路径
  Documents/file.txt     # 相对路径
  ./script.sh           # 当前目录下的相对路径
  ```

#### 通配符和扩展符号

**`*` (星号) - 零个或多个字符通配符**
- 含义：匹配零个或多个任意字符
- 使用规律：
  - `*.txt`：匹配所有以.txt结尾的文件
  - `file*`：匹配所有以file开头的文件
  - `*.log.*`：匹配所有包含.log.的文件
- 示例：
  ```bash
  ls *.txt              # 列出所有txt文件
  rm *.tmp              # 删除所有tmp文件
  cp *.jpg pictures/    # 复制所有jpg文件到pictures目录
  ```

**`?` (问号) - 单字符通配符**
- 含义：匹配任意单个字符
- 使用规律：
  - `file?.txt`：匹配file1.txt、file2.txt等（但不匹配file.txt或file10.txt）
  - `test_?.log`：匹配test_1.log、test_a.log等
- 示例：
  ```bash
  ls file?.txt          # 匹配file1.txt, file2.txt等
  rm temp_?.bak         # 删除temp_1.bak, temp_2.bak等
  ```

**`[abc]` (方括号) - 字符集合**
- 含义：匹配方括号中的任意一个字符
- 使用规律：
  - `[abc]`：匹配a、b或c
  - `[0-9]`：匹配任意数字
  - `[a-z]`：匹配小写字母
  - `[!abc]` 或 `[^abc]`：匹配除了a、b、c之外的字符
- 示例：
  ```bash
  ls file[123].txt      # 匹配file1.txt, file2.txt, file3.txt
  ls *[0-9].log         # 匹配所有以数字结尾的log文件
  ls [^a]*.txt          # 匹配不以a开头的txt文件
  ```

**`{a,b,c}` (花括号) - 花括号扩展**
- 含义：生成多个参数的组合
- 使用规律：
  - `file{1,2,3}.txt`：展开为file1.txt file2.txt file3.txt
  - `{jpg,png,gif}`：展开为jpg png gif
- 示例：
  ```bash
  cp file{1,2,3}.txt backup/    # 复制file1.txt, file2.txt, file3.txt
  mkdir -p project/{src,docs,tests}  # 创建src, docs, tests三个目录
  echo "Hello {World,Earth}"   # 输出两行：Hello World, Hello Earth
  ```

#### 转义和引用

**`\` (反斜杠) - 转义字符**
- 含义：取消特殊字符的特殊含义
- 使用规律：
  - `\*`：表示字面意义的星号，不是通配符
  - `\?`：表示字面意义的问号
  - `\$`：表示字面意义的美元符
- 示例：
  ```bash
  echo "Price is \$5.00"    # 输出: Price is $5.00
  touch file\ with\ spaces.txt  # 创建包含空格的文件名
  ls \*.txt                  # 查找名为*.txt的文件
  ```

**`" "` (双引号) 和 `''` (单引号)**
- 双引号：允许变量扩展和命令替换
- 单引号：保持字面含义，不进行任何扩展
- 示例：
  ```bash
  name="John"
  echo "Hello $name"         # 输出: Hello John
  echo 'Hello $name'         # 输出: Hello $name
  echo "Current dir: $(pwd)" # 输出当前目录
  echo 'Current dir: $(pwd)' # 输出字面字符串
  ```

### 💡 特殊符号使用技巧

1. **快速导航**
   ```bash
   cd ~          # 回主目录
   cd -          # 回上个目录
   cd ..         # 上一级目录
   cd ../..      # 上上级目录
   ```

2. **文件操作**
   ```bash
   cp ~/file.txt .           # 从主目录复制到当前目录
   mv file.txt ..            # 移动到上级目录
   ls -la ~/Documents        # 查看主目录的Documents
   ```

3. **批量操作**
   ```bash
   rm *.log                  # 删除所有log文件
   cp *.txt backup/          # 备份所有txt文件
   mkdir -p {src,bin,docs}   # 创建多个目录
   ```

4. **路径构造**
   ```bash
   # 使用相对路径
   ./script.sh               # 运行当前目录脚本
   ../tools/build.sh         # 运行上级目录的工具

   # 使用绝对路径
   /usr/bin/python script.py # 使用完整路径
   ```

### 1.1 目录浏览和切换

#### `ls` (list) - 列出目录内容
**核心概念：** 文件列表显示
- 基本用法：`ls`
- 常用选项：
  - `ls -l`：详细列表格式（权限、所有者、大小、时间）
  - `ls -a`：显示隐藏文件（以`.`开头的文件）
  - `ls -h`：人性化显示文件大小（K、M、G）
  - `ls -t`：按修改时间排序
- 示例：
  ```bash
  ls -lah    # 显示所有文件详细信息
  ls -lt     # 按时间倒序显示
  ```

#### `cd` (change directory) - 切换工作目录
**核心概念：** 目录导航
- 基本用法：`cd 目录路径`
- 特殊用法：
  - `cd ..`：上级目录
  - `cd ~` 或 `cd`：用户主目录
  - `cd -`：上一个目录
- 示例：
  ```bash
  cd /home/user/Documents
  cd ..    # 返回上级目录
  cd ~     # 返回主目录
  ```

#### `pwd` (print working directory) - 显示当前工作目录
**核心概念：** 定位当前目录
- 基本用法：`pwd`
- 常用选项：`pwd -P`（显示物理路径，解析符号链接）

### 1.2 文件和目录操作

#### `mkdir` (make directory) - 创建目录
**核心概念：** 目录创建
- 基本用法：`mkdir 目录名`
- 常用选项：
  - `mkdir -p`：递归创建目录（父目录不存在时自动创建）
- 示例：
  ```bash
  mkdir new_folder
  mkdir -p project/src/components  # 递归创建多层目录
  ```

#### `touch` - 创建空文件或更新时间戳
**核心概念：** 文件创建
- 基本用法：`touch 文件名`
- 特殊用法：如果文件已存在，只更新访问和修改时间

#### `cp` (copy) - 复制文件和目录
**核心概念：** 文件复制
- 基本用法：`cp 源文件 目标文件`
- 常用选项：
  - `cp -r`：递归复制目录
  - `cp -i`：交互式复制（覆盖前询问）
  - `cp -v`：显示复制过程
- 示例：
  ```bash
  cp file.txt backup.txt
  cp -r folder/ backup/
  cp -i *.txt documents/
  ```

#### `mv` (move) - 移动或重命名文件
**核心概念：** 文件移动和重命名
- 基本用法：`mv 源文件 目标文件`
- 常用选项：
  - `mv -i`：交互式移动
  - `mv -v`：显示移动过程
- 示例：
  ```bash
  mv old_name.txt new_name.txt  # 重命名
  mv file.txt documents/        # 移动到目录
  ```

#### `rm` (remove) - 删除文件和目录
**核心概念：** 文件删除
- 基本用法：`rm 文件名`
- 常用选项：
  - `rm -r`：递归删除目录
  - `rm -i`：交互式删除（删除前询问）
  - `rm -f`：强制删除（不询问）
- 示例：
  ```bash
  rm file.txt
  rm -rf folder/  # ⚠️ 危险操作，小心使用
  rm -i *.tmp     # 删除前询问
  ```

### 1.3 文件查看

#### `cat` (concatenate) - 查看文件内容
**核心概念：** 文件内容显示
- 基本用法：`cat 文件名`
- 常用选项：
  - `cat -n`：显示行号
- 示例：
  ```bash
  cat README.md
  cat -n script.sh
  ```

#### `less` 和 `more` - 分页查看文件
**核心概念：** 大文件分页浏览
- `less` 用法：
  - 空格/↓：向下翻页
  - b/↑：向上翻页
  - q：退出
  - /pattern：搜索
- 示例：`less large_file.log`

#### `head` 和 `tail` - 查看文件开头或结尾
**核心概念：** 文件内容预览
- 基本用法：
  - `head -n 10 file`：显示前10行
  - `tail -n 10 file`：显示后10行
- 特殊用法：
  - `tail -f file`：实时监控文件变化（日志监控）
- 示例：
  ```bash
  head -20 access.log
  tail -50 error.log
  tail -f /var/log/syslog
  ```
