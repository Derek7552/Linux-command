## 👥 贡献者指南

### 贡献流程
1. Fork项目到个人仓库
2. 创建功能分支 (`git checkout -b feature/功能描述`)
3. 提交更改 (`git commit -m "feat: 描述更改内容"`)
4. 推送分支 (`git push origin feature/功能描述`)
5. 创建Pull Request
6. 等待代码审查和合并

### 提交规范
遵循 [Conventional Commits](https://conventionalcommits.org/) 规范：

```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 重构
test: 测试相关
chore: 构建或工具配置
```

### 文档规范
- 使用Markdown格式
- 代码示例使用代码块并标注语言
- 保持中英文混用时的格式一致性
- 链接引用使用相对路径
